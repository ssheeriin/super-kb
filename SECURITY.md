# Security Policy

## Supported Versions

Security fixes are applied to:

- The latest released version
- The `main` branch for upcoming releases

Older releases may not receive patches.

## Reporting a Vulnerability

Please do not report security vulnerabilities in a public GitHub issue.

Preferred path:

1. Use GitHub's private vulnerability reporting for this repository if it is
   available in the repository UI.
2. If private reporting is not available, contact the maintainer privately via
   GitHub at `@ssheeriin` before public disclosure.

Include as much detail as possible:

- Affected version
- Operating system
- Reproduction steps
- Expected versus actual behavior
- Impact assessment
- Any proof-of-concept or logs that help reproduce the issue

You will receive an acknowledgement as soon as practical. Please allow time for
triage, reproduction, and patch preparation before public disclosure.
