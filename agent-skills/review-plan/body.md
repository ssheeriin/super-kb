# Review Plan

Use this skill when the user provides a plan and wants it checked against the
current `super-kb` implementation.

## Workflow

1. Read the plan document first.
2. Compare it with the actual repository structure and current user flows.
3. Check for gaps in:
   - runtime implementation
   - CLI and MCP behavior
   - provisioning and project setup
   - packaging and release assets
   - tests
   - documentation
4. Restructure the work into executable phases with verification points.

## Output

- Findings first.
- Then the revised phased plan.
